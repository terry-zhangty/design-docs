Mock Control CLI — Implementation Roadmap

A two-phase plan for building a CLI tool to send MockControlTrigger commands

Author: Tianyi Zhang Date: Feb 12, 2026 Status: Planning

1 Background

Problem

The Mock Control component in onboard_main accepts commands via the mock_control_trigger AOS/DDS topic. Currently, the only way to send these commands is through the APOV native UI, which cannot be operated remotely or scripted.

Goal

Build a command-line tool that can publish MockControlTrigger messages to the DDS topic, enabling scripted testing, remote operation, and CI/CD integration.

Key Interfaces

  • MockControlTrigger — protobuf message for sending commands (action, mode, actuator targets)
  • MockControlStatus — protobuf message published by the component with current state
  • mock_control_trigger — AOS/DDS topic name (publish)
  • mock_control_status — AOS/DDS topic name (subscribe)

4 Implementation Roadmap

Phase 1 — One-Shot CLI Tool

~1 hour

Build a fire-and-forget binary that sends a single MockControlTrigger message and exits. Immediately usable for scripting and manual testing.

  • Implement MockControlSender class (shared core): AOS ComponentNode init, topic publisher, message builder
  • Implement mock_ctl_main.cc: parse flags, call sender, exit
  • Define CLI flags: --action, --mode, --throttle, --brake, --steer, --profile
  • Write BUILD target for mock_ctl binary
  • Test with onboard_main in mock control mode on vcan
  • Verify actuator response via APOV or MockControlStatus logs

Deliverables

mock_ctl mock_control_sender.h/.cc BUILD

Phase 2 — Interactive REPL

Optional +1–2 hours

Extend the shared core with a persistent interactive CLI that includes a command loop and live status display. Build only if interactive debugging proves valuable.

  • Add REPL loop: readline, command parsing, dispatch to sender
  • Add DDS subscriber for mock_control_status on background thread
  • Implement live status formatter (throttle, brake, steer, mode display)
  • Implement command set: mode, start, stop, status, help, quit
  • Build interactive_mock_ctl binary
  • Test interactive session with live onboard_main

Deliverables

interactive_mock_ctl interactive_mock_ctl_main.cc

5 Usage Examples

Phase 1: One-Shot Commands

# Switch to auto driving mode
mock_ctl --action=ENABLE_ACTUATOR_OR_SWITCH \
         --mode=AUTO_MODE

# Start throttle at 50% (step profile)
mock_ctl --action=START_SIGNAL \
         --throttle=0.5 \
         --profile=step

# Stop all active signals
mock_ctl --action=STOP_SIGNAL

# Scripted test sequence
mock_ctl --mode=auto && sleep 2 && \
mock_ctl --action=START_SIGNAL \
         --throttle=0.3 && \
sleep 5 && \
mock_ctl --action=STOP_SIGNAL
