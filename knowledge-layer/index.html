Executive Summary

执行摘要 · 决策与请求一览

  • Problem · 问题 — Agent knowledge (rules, skills, methodology) is scattered across personal setups—no portable, sharable team baseline. · 知识分散在个人配置，缺少可共享的团队基线。
  • Proposal · 方案 — A structured Knowledge Layer following the agentskills.io open specification: three-layer progressive disclosure, cross-tool compatibility (Cursor, Codex, Claude Code), automated validation. · 三层渐进披露、跨工具兼容与自动化校验。
  • Ask · 请求 — Review 3 stacked PRs (~1200 lines total, split for focused review). MVP lands in 3 merges. · 审阅三个堆叠 PR，MVP 分三次合并。

Problem → Solution

问题与对策 · 从分散到结构化

Before 之前

  • Rules and guidance scattered; hard to share a single team baseline. · 规则与指引分散，难以形成统一的团队基线。
  • No common progressive-disclosure shape across tools. · 各工具缺少统一的 progressive disclosure 形态。
  • Personal or machine-specific notes do not travel as portable packs. · 个人或机器专属笔记难以作为可移植包流转。

After 之后

  • Structured domain packs under one workspace root with a thin cross-tool index. · 在工作区根目录下集中 domain pack，并配薄层跨工具索引。
  • L1→L2→L3 loading: catalog hint, SKILL.md router, references/ on demand. · L1→L2→L3 加载：目录提示、SKILL.md 路由、按需加载 references/。
  • Cross-tool entry: same packs for Cursor, Codex, and Claude Code conventions. · 跨工具入口：同一套包适配 Cursor、Codex、Claude Code 约定。

Why It Matters

业务价值 · 速度、质量与范围

  • Faster onboarding · 更快上手 — New team members get structured methodology instead of tribal knowledge. · 新人获得结构化方法，而非依赖口耳相传。
  • Quality guardrails · 质量护栏 — Automated checks catch secrets, broken references, and rule bloat before merge. · 合并前拦截密钥、断链与规则膨胀。
  • Cross-tool reach · 跨工具覆盖 — Same packs work for Cursor IDE, Codex CLI, and Claude Code without duplication. · 同一套包在多工具复用，避免重复维护。

Architecture Overview

三层渐进披露 · 与多工具入口对齐

L1
Catalog / discovery — description in SKILL.md frontmatter (~100 tokens). Stable routing surface: what/when/not.
目录 / 发现 — SKILL.md frontmatter 中的 description（约 100 tokens）。稳定路由面：做什么 / 何时 / 不做什么。
↓
L2
Pack body — SKILL.md content (<5k tokens / <500 lines): safety, defaults, pointers—not long tutorials.
包主体 — SKILL.md 正文（<5k tokens / <500 行）：安全、默认值、指针——非长篇教程。
↓
L3
Deep references — references/*.md loaded only when the router points there (guideline <300 lines each).
深度参考 — 仅当路由指向时加载 references/*.md（建议每篇 <300 行）。

Research Highlights

调研摘要 · 五条主线

Agent Skills Spec · Agent Skills 规范

  • Three layers: catalog → SKILL.md → references/ (agentskills.io model). · 三层：catalog → SKILL.md → references/（agentskills.io 模型）。
  • skills-ref validate rejects unknown top-level keys—extensions belong in metadata. · skills-ref validate 拒绝未知顶层键——扩展应放在 metadata。
  • 17 example skills in anthropics/skills; oversized SKILL.md is an anti-pattern signal. · anthropics/skills 提供 17 个示例；过大的 SKILL.md 是反模式信号。

Keep SKILL.md a router; depth in references + scripts. · 将 SKILL.md 作为路由；深度放在 references 与 scripts。

Cross-Tool Compat · 跨工具兼容

  • Thin root index + domain bodies to respect Codex merge limits. · 薄根索引 + domain 正文，以适配 Codex 合并上限。
  • Thin CLAUDE.md that delegates to shared index where appropriate. · 精简 CLAUDE.md，在合适处委托给共享索引。
  • Team marketplaces are org capabilities—not assumed for everyone. · Team marketplace 属于组织级能力——并非人人具备。

Thin AGENTS + pack SKILL bodies; avoid duplicating methodology at root. · 薄 AGENTS + 各 pack 的 SKILL 正文；避免在根目录重复方法论。

Security & Safety · 安全与合规

  • Secret/path leakage is a top risk in flow analysis. · 密钥/路径泄露是流程分析中的首要风险。
  • Non-negotiable policies must live in L2, not only L3. · 不可妥协的策略必须落在 L2，而非仅 L3。
  • Official anti-patterns: menu-style options, gotchas only in reference, kitchen-sink skills. · 官方反模式：菜单式选项、陷阱仅写在参考里、万能技能包。

Use blocking checklists in L2; plan gitleaks/trufflehog with CI hardening. · 在 L2 使用阻塞式清单；规划 gitleaks/trufflehog 与 CI 加固。

Governance · 治理

  • Pipeline pattern: skills-ref → secret scan → cross-ref → rule budget (staged). · 流水线：skills-ref → 密钥扫描 → 交叉引用 → 规则预算（分阶段）。
  • MVP: CONTRIBUTING + verify script; hardening: automated secret scanning. · MVP：CONTRIBUTING + verify 脚本；加固：自动化密钥扫描。
  • Lifecycle: MVP → hardening → long-term schema / multi-pack conflicts. · 生命周期：MVP → 加固 → 长期 schema / 多包冲突管理。

Land CONTRIBUTING + PR checklist with security and ownership early. · 尽早落地 CONTRIBUTING 与 PR 清单，明确安全与归属。

Patterns & Anti-patterns · 模式与反模式

  • Suggested flow: Before you start → Defaults → Procedure → When to load references. · 建议流程：开始前 → 默认值 → 步骤 → 何时加载 references。
  • Strong patterns: “Use when…”, quick-reference tables, deterministic scripts. · 强模式：「Use when…」、速查表、确定性脚本。
  • Avoid rule bloat and non-deterministic disclosure—hurts adoption. · 避免规则膨胀与非确定性披露——损害采纳。

Enforce consistent references/ naming; keep L2 skimmable. · 统一 references/ 命名；保持 L2 可扫读。

Key Design Decisions

关键设计决策 · 点击展开

Progressive disclosure — L2 holds non-negotiable safety 渐进披露：L2 承载不可妥协的安全要点
Safety policies must be visible in the SKILL.md body (L2), not only in deep references—use checklists and validation loops. L3 is for depth once the task matches a trigger.

安全策略必须在 SKILL.md 正文（L2）可见，而非仅存在于 deep references——使用清单与校验循环。任务匹配触发条件后，再由 L3 展开深度。
Metadata-only extensions for custom keys 自定义键仅走 metadata，保持校验器兼容
skills-ref validate rejects unknown top-level YAML keys; team extensions belong under metadata so CI stays aligned with upstream schema behavior.

skills-ref validate 拒绝未知顶层 YAML 键；团队扩展应放在 metadata，使 CI 与上游 schema 行为一致。
Team → Project → User merge order 规则合并优先级（Cursor 对齐）
When rules conflict, treat it as a bug. Personal overlays should only affect non-normative preferences—never weaken team-enforced rules. Documented in CONTRIBUTING with link to Cursor rules docs.

规则冲突应视为 bug。个人覆盖仅影响非规范性偏好——不得削弱团队强制规则。在 CONTRIBUTING 中说明并链接 Cursor rules 文档。
Bash-first verify script (minimal toolchain) Bash 校验脚本，工具链最小化
Stage A uses ./scripts/verify-public-pack.sh with bash, Python 3, and grep—optional ripgrep. Keeps local and future CI behavior aligned without heavy deps; npx skills-ref validate is a separate Stage B when Node is available.

Stage A 使用 ./scripts/verify-public-pack.sh，依赖 bash、Python 3 与 grep——可选 ripgrep。在轻依赖下对齐本地与未来 CI；npx skills-ref validate 在具备 Node 时作为独立 Stage B。
Separate AGENTS.md and CLAUDE.md entry points 分流入口：AGENTS 与 CLAUDE
Single cross-tool index in AGENTS.md; tool-specific bootstraps stay thin. CLAUDE.md delegates via @AGENTS.md so Claude Code routes to the same packs without duplicating long guidance.

AGENTS.md 承载单一跨工具索引；各工具引导保持精简。CLAUDE.md 通过 @AGENTS.md 委托，使 Claude Code 路由到同一套包而不重复冗长指引。
