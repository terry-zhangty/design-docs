Executive Summary

执行摘要 · 决策与请求一览

  • Problem · 问题 — Agent knowledge (rules, skills, methodology) is scattered across personal setups—no portable, sharable team baseline. · 知识分散在个人配置，缺少可共享的团队基线。
  • Proposal · 方案 — A structured Knowledge Layer following the agentskills.io open specification: three-layer progressive disclosure, cross-tool compatibility (Cursor, Codex, Claude Code), automated validation. · 三层渐进披露、跨工具兼容与自动化校验。
  • Ask · 请求 — Review 3 stacked PRs (~1200 lines total, split for focused review). MVP lands in 3 merges. · 审阅三个堆叠 PR，MVP 分三次合并。

Problem → Solution

问题与对策 · 从分散到结构化

Before 之前

  • Rules and guidance scattered; hard to share a single team baseline.
  • No common progressive-disclosure shape across tools.
  • Personal or machine-specific notes do not travel as portable packs.

After 之后

  • Structured domain packs under one workspace root with a thin cross-tool index.
  • L1→L2→L3 loading: catalog hint, SKILL.md router, references/ on demand.
  • Cross-tool entry: same packs for Cursor, Codex, and Claude Code conventions.

Architecture Overview

三层渐进披露 · 与多工具入口对齐

L1
Catalog / discovery — description in SKILL.md frontmatter (~100 tokens). Stable routing surface: what/when/not.
↓
L2
Pack body — SKILL.md content (<5k tokens / <500 lines): safety, defaults, pointers—not long tutorials.
↓
L3
Deep references — references/*.md loaded only when the router points there (guideline <300 lines each).

Cross-tool compatibility · 跨工具入口

Cursor
Rules precedence: Team → Project → User (official docs).
Codex
AGENTS.md merged default 32 KiB ceiling; root→leaf, later overrides.
Claude Code
CLAUDE.md guidance <200 lines; supports @import and nested files.

Cursor CLI does not load IDE plugins—do not assume plugin-only flows in headless automation.

Research Highlights

调研摘要 · 五条主线

Agent Skills Spec

  • Three layers: catalog → SKILL.md → references/ (agentskills.io model).
  • skills-ref validate rejects unknown top-level keys—extensions belong in metadata.
  • 17 example skills in anthropics/skills; oversized SKILL.md is an anti-pattern signal.

Keep SKILL.md a router; depth in references + scripts.

Cross-Tool Compat

  • Thin root index + domain bodies to respect Codex merge limits.
  • Thin CLAUDE.md that delegates to shared index where appropriate.
  • Team marketplaces are org capabilities—not assumed for everyone.

Thin AGENTS + pack SKILL bodies; avoid duplicating methodology at root.

Security & Safety

  • Secret/path leakage is a top risk in flow analysis.
  • Non-negotiable policies must live in L2, not only L3.
  • Official anti-patterns: menu-style options, gotchas only in reference, kitchen-sink skills.

Use blocking checklists in L2; plan gitleaks/trufflehog with CI hardening.

Governance

  • Pipeline pattern: skills-ref → secret scan → cross-ref → rule budget (staged).
  • MVP: CONTRIBUTING + verify script; hardening: automated secret scanning.
  • Lifecycle: MVP → hardening → long-term schema / multi-pack conflicts.

Land CONTRIBUTING + PR checklist with security and ownership early.

Patterns & Anti-patterns

  • Suggested flow: Before you start → Defaults → Procedure → When to load references.
  • Strong patterns: “Use when…”, quick-reference tables, deterministic scripts.
  • Avoid rule bloat and non-deterministic disclosure—hurts adoption.

Enforce consistent references/ naming; keep L2 skimmable.

Key Design Decisions

关键设计决策 · 点击展开

Progressive disclosure — L2 holds non-negotiable safety 渐进披露：L2 承载不可妥协的安全要点
Safety policies must be visible in the SKILL.md body (L2), not only in deep references—use checklists and validation loops. L3 is for depth once the task matches a trigger.
Metadata-only extensions for custom keys 自定义键仅走 metadata，保持校验器兼容
skills-ref validate rejects unknown top-level YAML keys; team extensions belong under metadata so CI stays aligned with upstream schema behavior.
Team → Project → User merge order 规则合并优先级（Cursor 对齐）
When rules conflict, treat it as a bug. Personal overlays should only affect non-normative preferences—never weaken team-enforced rules. Documented in CONTRIBUTING with link to Cursor rules docs.
Bash-first verify script (minimal toolchain) Bash 校验脚本，工具链最小化
Stage A uses ./scripts/verify-public-pack.sh with bash, Python 3, and grep—optional ripgrep. Keeps local and future CI behavior aligned without heavy deps; npx skills-ref validate is a separate Stage B when Node is available.
Separate AGENTS.md and CLAUDE.md entry points 分流入口：AGENTS 与 CLAUDE
Single cross-tool index in AGENTS.md; tool-specific bootstraps stay thin. CLAUDE.md delegates via @AGENTS.md so Claude Code routes to the same packs without duplicating long guidance.
